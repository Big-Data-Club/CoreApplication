Part 2 · Mastery Modeling

Bayesian Knowledge Tracing (BKT)

Why BKT?
Standard recommenders know what you visited. BKT models whether you understood it. It tracks the hidden state of concept mastery based on correct/incorrect response sequences.
Step-by-Step BKT Update Mechanics
1
Fetch Prior: P(L_t-1)
Initial probability of mastery (starts at P(L0) parameter).
2
Assess action outcome (correct/incorrect)
Student attempts a check. Correct counts map to probability updates.
3
Compute Posterior P(L_t-1 | Action)
Applies Bayes' theorem using slip P(S) and guess P(G) factors.
4
Apply learning transition P(T)
Accounts for learning event to yield final mastery state P(L_t).
BKT Probability Equations
$$P(L_{t-1} \mid \text{Correct}) = \frac{P(L_{t-1}) \cdot (1 - P(S))}{P(L_{t-1}) \cdot (1 - P(S)) + (1 - P(L_{t-1})) \cdot P(G)}$$
$$P(L_{t-1} \mid \text{Incorrect}) = \frac{P(L_{t-1}) \cdot P(S)}{P(L_{t-1}) \cdot P(S) + (1 - P(L_{t-1})) \cdot (1 - P(G))}$$
$$P(L_t) = P(L_{t-1} \mid \text{Action}) + (1 - P(L_{t-1} \mid \text{Action})) \cdot P(T)$$
Downstream RS Applications
  • P(mastery) < 0.8 $\to$ Flags knowledge gap $\to$ RS boosts practice items
  • P(mastery) ≥ 0.8 $\to$ Mastery reached $\to$ System unlocks next lesson node
  • Performance decline $\to$ Triggers Spaced Repetition review
  • Explain Bayesian updates of mastery state after check attempts.
  • Relate prior variables and transitional probabilities to learning trajectories.
"Bayesian Knowledge Tracing updates the student's mastery probability dynamically. By estimating guess rates and slip rates, we calculate the posterior probability after correct or incorrect answers, deciding whether to serve review items."

Part 3 · Advanced Algorithms

Sequential Recommendation

Why Sequential Models?
Learning order is strict: A $\to$ B $\to$ C differs from B $\to$ A $\to$ C. Sequential recommender models treat user history as an ordered sequence of events to predict the next action.
GRU4Rec Forward Path Step-by-Step
1
Represent history sequence
Input sequence of item IDs: $[i_1, i_2, ..., i_{t-1}]$.
2
Project to Embedding Space
Embeds each item ID into a dense vector representation.
3
Process with recurrent cells
Feeds embedded vectors into GRU cells to accumulate sequential dependencies.
4
Map hidden state to logits
Decodes last hidden vector $h_{t-1}$ to calculate next item vocabulary logits.
class GRU4RecBackbone(nn.Module):
    def __init__(self, vocab_size, hidden=128):
        super().__init__()
        self.item_emb = nn.Embedding(vocab_size, 64)
        self.gru = nn.GRU(64, hidden, num_layers=2)
        self.fc = nn.Linear(hidden, vocab_size)

    def forward(self, seq):
        x = self.item_emb(seq)
        h, _ = self.gru(x)
        logits = self.fc(h[:, -1, :])
        return logits
SRD Composite Loss Equation
$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{CE}} + \lambda \cdot \mathcal{L}_{\text{diversity}}$$

Part 3 · Advanced Algorithms

Reinforcement Learning Fine-Tuning

RL Tuning Paradigm
Pre-train sequential models offline. Then, treat recommendation slates as actions and update the policy using REINFORCE gradient updates based on user success rewards.
REINFORCE Policy Update Step-by-Step
1
Run policy forward pass
Generate model logits $\pi_\theta(a|s)$ representing probability values over catalog items.
2
Sample K actions
Sample a slate of Top-K recommendation items using multinomial sampling.
3
Acquire environmental reward R
Measure user engagement: reward = nDCG@K + 0.2 × item novelty.
4
Compute loss & update weights
Update network weights using gradient ascent: $\theta \leftarrow \theta + \alpha R \nabla \log \pi_\theta(a|s)$.
Policy Gradient (REINFORCE) Equation
$$\nabla_\theta J(\theta) = \mathbb{E} \left[ \nabla_\theta \log \pi_\theta(a \mid s) \cdot R(a,s) \right]$$ 
def train_rl_step(model, ref_model, batch):
    logits = model(batch['seq'])
    actions = sample_top_k(logits, k=5)
    rewards = compute_rewards(actions, batch)

    # KL Divergence constraint (DivKG)
    kl = kl_div(logits, ref_model(batch))
    loss = -(rewards * logits).mean() + 0.05 * kl
    
    loss.backward(); optimizer.step()

Part 3 · Memory Retention

Spaced Repetition Systems (SRS)

🧬 Ebbinghaus Forgetting Curve
Memory retention decays exponentially. Spaced repetition calculates the optimal review date — right before the user is likely to forget — to lock knowledge in long-term memory.
SuperMemo-2 Algorithm Step-by-Step
1
Collect quality score (0-5)
User rates recall performance on a flashcard.
2
Adjust Easiness Factor (EF)
Update easiness factor based on rating value.
3
Calculate next interval (I)
Calculates spacing interval in days for next repetition.
4
Schedule next review date
Set next_review_date to current time + interval days.
SM-2 Spacing Formulas
$$EF' = EF + (0.1 - (5 - q) \cdot (0.08 + (5 - q) \cdot 0.02))$$
$$I(n) = I(n-1) \cdot EF$$

Part 4 · Validation Design

Quy trình Temporal Evaluation

⚠️ Preventing Data Leakage
Never split recommendation datasets randomly. Because learning behavior is chronological, random splits leak future interactions. Always partition using a Temporal Split.
Temporal Split Protocol Step-by-Step
1
Sort user interactions chronologically
Order all logged student actions by their timestamp field.
2
Establish division boundary
Define a boundary cut point (e.g. last 20% of actions per user).
3
Train on past logs
Train recommendation algorithms exclusively on pre-split historical data.
4
Evaluate predictions on future events
Test predictions against post-split records (guarantees zero look-ahead bias).
Implementation Code Pattern
def temporal_split(df, test_ratio=0.2):
    # Sort chronologically
    df = df.sort_values('timestamp')
    # Slice the last 20% records per user
    test_idx = df.groupby('user_id').apply(
        lambda g: g.tail(int(len(g) * test_ratio))
    ).index.get_level_values(1)
    
    train_set = df.drop(test_idx)
    test_set = df.loc[test_idx]
    return train_set, test_set
